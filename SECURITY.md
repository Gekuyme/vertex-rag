# Security Policy

## Reporting a vulnerability

If you discover a security issue in Vertex RAG, please do not open a public issue with exploit details.

Instead:

1. Prepare a short report with affected components, impact, reproduction steps, and suggested mitigation if known.
2. Share it privately with the maintainers.
3. Allow reasonable time for triage, remediation, and coordinated disclosure.

## What to include

- affected version or commit
- environment details
- exact reproduction steps
- proof of impact
- whether credentials, tenant data, or document access boundaries are involved

## Priority areas

Security-sensitive areas in this project include:

- authentication and token handling
- role-based access control
- document visibility boundaries
- file upload and ingestion pipeline
- storage integration
- web-search augmentation
- multi-tenant data isolation

## Disclosure expectations

We will aim to acknowledge reports promptly, validate impact, and ship a fix or mitigation before public disclosure where possible.
